## 🔍 [2단계 작업: 누락 복구 + 스타일 교정]

### 📋 1단계: 누락 콘텐츠 복구
마크다운 원본 파일(쇼츠학개론 정규강의 1주차 1강 2부.md)과 현재 구현된 컴포넌트를 대조하여 누락된 정보를 추가하십시오.

#### 🛠️ 작업 조건:
1. **기존 시각화 보존**: 이미 구현된 훌륭한 UI/디자인 스타일은 절대 건드리지 마십시오.
2. **스타일 일관성**: 추가되는 내용 역시 기존 시각화와 동일한 패턴(컴포넌트, 색상 테마, 들여쓰기)을 유지하십시오.
3. **디테일 100% 복구**: 마크다운 원본의 모든 상세 정보(숫자, 조건, 예외사항, 플랫폼 목록 등)를 생략 없이 포함하십시오.

#### ⚠️ 예상 누락 유형 (반드시 체크):
- **상세 조건/예외사항**: (예: "단, ~인 경우는 X")
- **구체적인 숫자/기준**: (예: "3,600만 원 이상 ~ 7,500만 원 미만")
- **추가 플랫폼/서비스 목록**: (예: 생략된 도구나 사이트 이름)
- **계산식/공식 및 세부 절차**: (예: "A x B = C" 같은 공식이나 구체적인 단계)


### 🎨 2단계: 기존 스타일 교정 
누락 복구와 **동시에**, 기존 코드에서 프롬프트에 맞지 않는 스타일을 모두 찾아 수정하십시오.

#### 🔴 반드시 찾아서 수정해야 할 잘못된 스타일:

말아하기 -> 마라하기
| 잘못된 스타일 (찾아서 제거) | 올바른 스타일 (변경 후) |
|:---|:---|
| `text-xs`, `text-[10px]`, `text-[0.65rem]`, `text-[0.7rem]` | `text-base`, `text-lg`, `text-xl` |
| `text-zinc-400`, `text-zinc-500`, `text-zinc-600` | `text-zinc-100`, `text-zinc-200`, `text-zinc-300` |
| `**마크다운 강조**` (미적용 상태) | `<span className="text-white font-bold">텍스트</span>` |
| `p-4`, `p-5` (작은 패딩) | `p-6`, `p-8` (큰 패딩) |
| 아이콘 크기 미지정 | `size=${24}` 또는 `size=${28}` 명시 |

#### 🔍 특별 주의 영역:
- **카드 내부 텍스트**: 카드 안의 설명(`<p>`) 태그가 작은 글씨인지 확인
- **배지/라벨 텍스트**: `text-[10px]` 같은 극소 크기 사용 여부 확인
- **리스트 항목**: `<li>` 내부 텍스트가 어두운 회색인지 확인
- **보조 설명**: 팁, 주의사항, 보조 정보가 읽기 어려운 크기인지 확인

## 💎 디자인 시스템 준수 사항
추가하거나 수정하는 모든 텍스트는 아래의 규격을 엄격히 따릅니다.

1. **타이틀 (H1, H2)**: `font-black tracking-tighter`, 테마 컬러 그라디언트 및 이탤릭 적용.
2. **배지 (Badge)**: 최상단 `<${Badge} color="...">공식 시리즈 [N]<//>` 확인.
3. **히어로 설명**: `text-xl md:text-2xl text-white`, `border-l-4` 포인트 필수.
4. **본문 텍스트 (Body - 치명적)**: 
   - **절대 금지**: `text-xs`, `text-sm`, `text-zinc-500`(어두운 회색) 사용 금지.
   - **표준**: `text-lg lg:text-xl text-zinc-100 leading-relaxed` 규격 준수.
5. **강조**: `<span className="text-[테마컬러] font-bold">` 또는 `text-white font-black` 사용.
6. **아이콘**: 반드시 `size=${24}` 이상 명시.

---

## 🏗️ 콘텐츠 시각화 매핑 규칙
- **통계/숫자**: `text-5xl lg:text-7xl font-black`으로 압도적 강조.
- **과정/단계**: 번호 아이콘(`w-16 h-16`)과 화살표를 활용한 프로세스화.
- **불렛/리스트**: 단순 점(`•`) 지양, 개별 독립 카드로 분리하여 시각화.
- **인용구/팁**: `bg-zinc-800/50` 배경과 테두리 광채(`glow-[color]`) 효과 적용.
