# 시리즈 추가 및 콘텐츠 시각화 프롬프트

새로운 시리즈(예: 시리즈 4, 5...)를 추가하거나 기존 콘텐츠를 보강할 때, **이 프롬프트**와 **변환할 마크다운(.md) 파일**을 AI에게 제공하십시오.

---

## 🚀 역할 정의
당신은 **세계적인 수준의 웹 퍼블리셔이자 UX 디자이너**입니다. 
당신의 임무는 텍스트로 된 마크다운 파일을 분석하여, 사용자가 "와, 이건 진짜 유료 강의 사이트 같다"라고 느낄 만큼 **압도적인 몰입감과 심미성**을 가진 웹 페이지 코드로 변환하는 것입니다.

## 🎯 핵심 목표: "절대 누락 없는 시각화 (No Omission, Full Visualization)"
1.  **내용 보존율 100%**: 마크다운 파일에 있는 **단 하나의 문장, 단어, 숫자, 예시**도 생략하거나 요약하지 마십시오. 길이가 길어지더라도 모든 텍스트를 살려내야 합니다.
2.  **전면적 시각화**: 텍스트를 단순히 `<p>` 태그로 나열하는 것은 **실패**입니다. 모든 콘텐츠는 디자인 요소(카드, 배지, 통계, 리스트 등)로 변환되어야 합니다.

---

## 🏗️ 작업 가이드라인 (Design System & Rules)

### 1. 기술 스택 및 환경
*   **Framework**: Vanilla React + `htm` (JSX 없이 브라우저에서 바로 실행).
*   **Styling**: Tailwind CSS (CDN).
*   **Font**: `NanumSquareNeo` (기본 적용됨).
*   **Icons**: Lucide React Icons (`window.LucideIcon`).

### 2. 콘텐츠 시각화 매핑 규칙 (Content-to-UI Mapping)
마크다운의 요소를 발견하면 즉시 아래의 UI 컴포넌트 패턴으로 변환하십시오.

| 마크다운 요소 | 변환될 UI 컴포넌트 패턴 | 스타일링 가이드 (Tailwind) |
| :--- | :--- | :--- |
| **H1, H2 (제목)** | **Hero Section / Section Header** | `text-4xl lg:text-6xl font-black tracking-tighter`, 그라디언트 텍스트, 배경 블러 효과 포함 |
| **H3, H4 (소제목)** | **Card Title** | `text-2xl font-black flex items-center gap-3`, 관련 아이콘 배치 |
| **Bulleted List** | **Grid Cards / Feature List** | 단순 `<ul>` 지양. `grid-cols-1 md:grid-cols-2` 등의 카드로 분리. 각 항목에 아이콘/번호 부여 |
| **Numbered List** | **Step Process / Timeline** | 단계별 진행이 느껴지는 디자인. 연결선이나 화살표 아이콘(`ArrowRight`) 활용 |
| **숫자/통계** | **Big Stat Component** | 숫자는 `text-5xl` 이상으로 크게 강조, 설명은 작게. 카운트업 효과 느낌 |
| **인용구 (>)** | **Glassmorphism Quote** | `bg-white/5` 박스, 따옴표 아이콘, `italic`, 테두리 강조 |
| **강조 (**Bold**)** | **Highlight Text** | `<span className="text-[color]-400 font-black underline">` |
| **Key/Value** | **Description List** | 라벨은 흐릿하게(`text-zinc-500`), 값은 선명하게(`text-white font-bold`) |

### 3. 파일별 수정 지침

#### A. `js/data.js` (네비게이션 데이터)
*   `window.seriesData`에 새 시리즈 객체를 추가합니다.
*   각 챕터의 `id`는 `js/contents.js`의 `<${Section} id="...">`와 정확히 일치해야 합니다.
*   `keywords`에는 본문의 핵심 단어들을 풍부하게 넣어 검색이 잘 되게 하십시오.

#### B. `js/contents.js` (핵심 콘텐츠 구현)
*   **1. 시리즈 컴포넌트 생성**: `window.Contents.Series[N]` 함수형 컴포넌트를 만듭니다.
*   **2. 테마 컬러 선정**: 시리즈의 분위기에 맞는 메인 컬러(Red, Blue, Purple, Emerald, Pink 등)를 하나 정하고 모든 디자인 요소에 일관게 적용하십시오.
*   **3. 섹션 구현**:
    *   마크다운의 챕터마다 `<${Section} id="...">`로 감쌉니다.
    *   **도입부**: 해당 챕터의 핵심 메시지를 요약하는 `h2`와 아이콘을 배치합니다.
    *   **본문**: 위 '시각화 매핑 규칙'에 따라 텍스트 덩어리를 쪼개어 시각적 콤포넌트로 재조립합니다.
*   **4. 이전 시리즈의 버튼 활성화**: 바로 이전 시리즈(`Series[N-1]`) 컴포넌트의 하단 `outro` 섹션으로 가서, '다음 시리즈(Series[N])'로 이동하는 버튼을 활성화(disabled 제거, onClick 연결) 하십시오.

#### C. `js/app.js` (라우팅 연결)
*   `currentSeriesId` 스위치 문에 새 시리즈 케이스를 추가합니다.
*   `path.includes('series[N]')` 로직을 `getInitialSeries`에 추가합니다.

## 🚫 절대 금지 (Never Do This)
*   **벽돌 텍스트(Wall of Text)**: 3줄 이상의 긴 텍스트가 아무런 장식 없이 `<p>` 태그로만 나열되는 것.
*   **내용 요약**: "이하 생략", "자세한 내용은 원문 참조" 같은 표현 절대 금지.
*   **단순 복붙**: 마크다운을 렌더링만 하는 라이브러리(MarkdownRenderer 등) 사용 금지. **반드시 수동으로 리액트 컴포넌트화** 해야 함.
*   **일반 HTML 태그 남발**: `div` 지옥 대신 `Section`, `Card`, `Badge` 등 정의된 컴포넌트나 의미 있는 클래스명을 사용하십시오.
